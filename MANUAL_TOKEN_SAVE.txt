===============================================
QUICK FIX - Manually Save Token
===============================================

Your login component is NOT saving the token to localStorage.
Here's how to manually save it so you can test immediately:

STEP 1: Open Browser Console
------------------------------
Press F12 in your browser to open Developer Tools
Go to the "Console" tab

STEP 2: Run This Code
------------------------------
Copy and paste this entire code block into the console:

// Replace with YOUR email and password
const email = 'your-email@example.com';
const password = 'your-password';

fetch('http://localhost:8080/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email, password })
})
.then(response => response.json())
.then(data => {
  if (data.token) {
    localStorage.setItem('token', data.token);
    console.log('✅ Token saved successfully!');
    console.log('Token:', data.token.substring(0, 50) + '...');
    alert('Token saved! Now refresh the page and go to Raise Ticket.');
  } else {
    console.error('❌ No token in response:', data);
  }
})
.catch(error => {
  console.error('❌ Login failed:', error);
  alert('Login failed. Check console for details.');
});

STEP 3: Verify Token is Saved
------------------------------
Run this in console:

localStorage.getItem('token')

You should see a long string starting with "eyJ..."

STEP 4: Test Raise Ticket
------------------------------
1. Refresh the page (F5)
2. Go to Raise Ticket page
3. Error should be gone!
4. Phone should auto-fill
5. You can submit tickets

===============================================
PERMANENT FIX
===============================================

To fix this permanently, update your login component:

Add this line after successful login:
localStorage.setItem('token', response.data.token);
